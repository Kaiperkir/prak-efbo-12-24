## 📝 Описание проекта
Данный проект представляет собой полноценное веб-приложение интернет-магазина. Работа объединяет в себе материалы практических занятий №7–11, включая серверную часть на Express и клиентскую на React.

### Ключевой функционал:
*   **Безопасность:** Регистрация и аутентификация с использованием **JWT (Access & Refresh tokens)**.
*   **Хеширование:** Хранение паролей в зашифрованном виде через `bcrypt`.
*   **RBAC (Role-Based Access Control):** Система прав доступа для ролей: *Гость, Пользователь, Продавец, Администратор*.
*   **Интерцепторы:** Автоматическое обновление токенов на фронтенде через `Axios Interceptors`.
*   **Хранение данных:** Использование JSON-файлов в качестве базы данных.

## 🔐 Ролевая модель (RBAC)

| Роль | Маршруты | Доступные действия |
| :--- | :--- | :--- |
| **Гость** | `/login`, `/register` | Только вход и регистрация |
| **Пользователь** | `/products`, `/auth/me` | Просмотр товаров и своего профиля |
| **Продавец** | `/products` (POST/PUT) | Добавление и редактирование товаров |
| **Администратор** | `/users`, `/products` (DELETE) | Управление пользователями и удаление товаров |

---

## 🛰 Основные API Endpoints

### Auth
- `POST /api/auth/register` — Регистрация
- `POST /api/auth/login` — Вход (выдача пары токенов)
- `POST /api/auth/refresh` — Обновление токенов
- `GET /api/auth/me` — Данные текущего пользователя

### Products
- `GET /api/products` — Получить список всех товаров
- `POST /api/products` — Создать товар (Seller/Admin)
- `PUT /api/products/:id` — Обновить товар (Seller/Admin)
- `DELETE /api/products/:id` — Удалить товар (Admin)
